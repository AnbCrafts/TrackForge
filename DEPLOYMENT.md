#  Cloud Run Deployment Documentation: TrackForge

This document outlines the configurations, automated build pipelines (`cloudbuild.yaml`), and critical domain resolution steps required to successfully launch **TrackForge** on Google Cloud Platform (GCP).

---  

##  System Architecture on GCP

TrackForge is deployed as two independent, fully containerized microservices hosted on **Google Cloud Run**:

1. **Backend Service (`trackforge-server`)**
   - **Framework:** Express / Node.js
   - **Internal Port:** `9000`
   - **URL:** Automatically generated by GCP or mapped to a custom domain (e.g., `api.Hansrajiinfotech.com`).

2. **Frontend Service (`trackforge-client`)**
   - **Framework:** React (Vite) + Nginx
   - **Internal Port:** `80` (HTTP default)
   - **URL:** Automatically generated by GCP or mapped to a custom domain (e.g., `trackforge.Hansrajiinfotech.com`).

##  CRITICAL DEPLOYMENT & NETWORKING ISSUES (READ FIRST)

Before proceeding, please review these key resolutions regarding your subdomains, local hosts, and media permissions:

### 1.  The Local Hosts Workaround vs. Global DNS Propagation
> [!WARNING]
> **Current Status:** IP mapping for your subdomains is **still pending** on the global internet.
- **Why it worked for you:** You modified your local `hosts` file using Notepad (Run as Admin) to manually map `subdomaina.Hansrajiinfotech.com` to a specific IP address. This is a local DNS override.
- **Why it won't work for others:** The rest of the internet does not have your local `hosts` configuration. They will get a `DNS_PROBE_FINISHED_NXDOMAIN` error.
- **Action Required:** Your manager or IT Administrator must log into their domain registrar (e.g., GoDaddy, Cloudflare, Namecheap) or Cloud DNS and configure:
  - **A Record** (for mapping directly to a static IP address).
  - **CNAME Record** (for pointing your subdomain directly to the Google Cloud Run URL).

---

### 2.  Camera & Mic Permission Restrictions (HTTP vs. HTTPS)
> [!CAUTION]
> **Issue:** Modern web browsers (Chrome, Safari, Edge, Firefox) enforce strict security contexts for hardware APIs. Camera, Microphone, Geolocation, and Screen Recording **will be blocked on any insecure origin (`http://`)**.
- **The Problem:** Running your site on `http://subdomaina.Hansrajiinfotech.com` prevents the camera and mic from opening because it lacks SSL encryption.
- **The Resolution:** You **MUST** serve the frontend over `https://`.
- **How GCP handles this:** When you map your custom subdomain to Google Cloud Run, **GCP automatically provisions and renews a free Google-managed SSL Certificate (`https://`)** for your subdomains. 
- Once DNS mapping is complete, accessing `https://subdomaina.Hansrajiinfotech.com` will immediately restore camera and microphone capabilities.

##  How to Deploy (Step-by-Step)

### Step 1: Install & Initialize Google Cloud SDK
If you haven't installed it already, install the [Google Cloud CLI](https://cloud.google.com/sdk/docs/install) and run:
```bash
gcloud auth login
gcloud config set project [YOUR_GCP_PROJECT_ID]
gcloud services enable run.googleapis.com artifactregistry.googleapis.com cloudbuild.googleapis.com
```

### Step 2: Trigger the Automated Deployment File (`cloudbuild.yaml`)
Run the following command from the root directory of your project. GCP Cloud Build will read the configuration file, compile both applications, push them to the registry, and deploy them:
```bash
gcloud builds submit --config cloudbuild.yaml .
```

### Step 3: Map Your Subdomains & Enable SSL (GCP Cloud Run Console)
Once deployed successfully, map your domain globally to Cloud Run so it works for the entire internet:
1. Open the **Cloud Run Console** in GCP.
2. Click **Manage Custom Domains** from the top action bar.
3. Click **Add Mapping**:
   - Select the Cloud Run service (e.g., `trackforge-client`).
   - Choose **Specify a custom domain** and enter your subdomain (e.g., `subdomaina.Hansrajiinfotech.com`).
4. Click **Continue**. GCP will provide you with a **DNS Resource Record** (either an `A` record or a `CNAME` record).
5. **Send these records to your manager/IT Admin** to add them to the domain's DNS panel.
6. Once the DNS propagates, GCP will automatically issue a **Let's Encrypt / Google Trust Services SSL Certificate (`https://`)** within 10–30 minutes, resolving your camera and microphone restriction issues!
